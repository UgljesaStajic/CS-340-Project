## Project Summary

This project is an interactive dashboard created for Grazioso Salvare. The dashboard connects to the Austin Animal Center Outcomes dataset in MongoDB and helps users find dogs that match different rescue-training categories.

The dashboard includes the required Grazioso Salvare logo, creator name, rescue filters, interactive data table, pie chart, and geolocation map.

## Features

- Connects to the `aac.animals` collection in MongoDB.
- Uses the Project One CRUD Python module to read animal outcome records.
- Provides rescue-category filters for targeted dog searches.
- Updates the data table based on the selected filter.
- Updates a pie chart showing the top 10 breeds from the current results.
- Updates a map marker using the selected animal's latitude and longitude.
- Includes a reset option to return the dashboard to its starting state.

## Rescue Filter Options

The dashboard includes four filter options:

1. Water Rescue
2. Mountain or Wilderness Rescue
3. Disaster or Individual Tracking
4. Reset

When a filter is selected, the dashboard updates the table, breed pie chart, and geolocation map together.

## Tools and Technologies Used

- Python
- MongoDB
- PyMongo
- Dash / JupyterDash
- Plotly Express
- Dash Leaflet
- JupyterLab

MongoDB was used because it stores the animal shelter data as documents and works well with Python through PyMongo. Dash was used to create the dashboard layout and callbacks. The callbacks allow the rescue filters, data table, pie chart, and map to update together.

## How to Run

1. Make sure MongoDB is running.
2. Import the Austin Animal Center Outcomes dataset into the `aac.animals` collection.
3. Open `ProjectTwoDashboard.ipynb` in JupyterLab.
4. Run the notebook from the top.
5. Open the Dash link generated by the notebook.
6. Test each rescue filter and the reset option.

## Development Process

I started the assignment by reviewing all requirements and documents so the dashboard would match the requested functionality. I used the provided `ProjectTwoDashboard.ipynb` template and connected it to MongoDB through my CRUD module.

After the database connection was working, I added the Grazioso Salvare logo and creator name. I then added the rescue filters, connected the filters to MongoDB queries, and built the data table, pie chart, and map.

At the end, I tested each filter and took screenshots to confirm the dashboard worked correctly.

## Challenges and Solutions

One challenge was that the pie chart became crowded when it displayed too many breeds. I fixed this by limiting the chart to the top 10 breeds from the current results.

Another challenge was preventing callback errors when no animal was selected. I fixed the callback logic so the dashboard would not crash when the selected row was empty.

A final challenge was the layout of the pie chart and map. They looked squeezed when displayed side by side, so I adjusted the map width to make the dashboard easier to read.
